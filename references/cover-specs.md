# 多平台封面规格 · Cover Specs

用于基于一份文章 / deck 的核心结论生成**独立单张封面**（不是 deck 里的一页 slide）。常见落位：公众号头图、公众号分享卡、小红书封面、视频号横版封面、抖音/小红书视频竖封。

封面和 deck 共用同一套**专业咨询风**视觉系统（McKinsey 调色板 + 无衬线 + 结论先行），但**绝不要直接截 deck 的某一页当封面**——deck 横向版式信息密度低，缩到 1:1 或 3:4 会留白过多、字号失衡。封面必须**重新构图**。

## 何时走这条路径

- 用户说"做一张封面 / 头图 / 分享卡 / 小红书图 / 视频号封面"
- 用户已经有一份 deck 或一篇文章，需要一张配套发图
- 用户要在同一主题下生成多个尺寸的封面（发多个平台）

**不适用**：封面里要嵌很多文字段落（那是海报，不是封面）；需要可编辑分层 PSD；需要动图。

## 平台规格表

| 平台 | 比例 | 像素建议 | 安全区位置 | 典型用途 |
|---|---|---|---|---|
| 公众号头图 | **21:9** | 2400 × 1029 | 顶部 / 左侧，**避开右下角公众号水印** | 顶部 banner |
| 公众号分享卡 | **1:1** | 1080 × 1080 | 中央 60% | 朋友圈 / 群分享缩略图 |
| 小红书封面 | **3:4** | 1080 × 1440 | 上 60%，**避开底部用户名 + 标签条** | 信息流首图 |
| 小红书轮播图 | **3:4** | 1080 × 1440 | 同上，系列内统一字号 | 多图笔记 |
| 视频号横版封面 | **16:9** | 1920 × 1080 | 中央 70%，**避开右下时长水印** | 视频号 / B 站 / YouTube 封面 |
| 抖音 / 小红书视频竖封 | **9:16** | 1080 × 1920 | 上 1/3 标题，中部主视觉，**避开底部进度条** | 短视频封面 |

**额外约束**：
- 不允许自定义任意尺寸——只能从上表选
- 同主题多平台时，**主标题 + 主题色 + 视觉锚点保持一致**，只改构图比例和字号
- 像素至少满足建议值，避免上传后压缩糊掉

## 视觉系统（与 deck 一致 · 不允许自定义）

**配色（McKinsey 标准调色板，hex 锁死）**：

| 角色 | hex | 用途 |
|---|---|---|
| `--paper` | `#ffffff` | 主背景（亮底封面） |
| `--ink` | `#17212B` | 主标题 / 正文（亮底） |
| `--paper-tint` | `#f4f6f8` | 浅灰背景块 |
| `--ink-dark` | `#0a0e13` | 暗底封面背景（dark 变体） |
| `--paper-on-dark` | `#ffffff` | 暗底封面文字 |
| `--navy` | `#17365D` | 主蓝（数据 / 主色块） |
| `--red` | `#C62828` | 单点 accent（红色 vertical 钉 / 关键数字） |
| `--green` | `#3A7D5A` | 正向数据 / 状态色 |
| `--amber` | `#D79B35` | 中等数据 / 状态色 |

**字体**：Inter / Helvetica / 苹方 / 雅黑 / Noto Sans SC（无衬线 only）

**装饰元素**：
- 1px hairline 分隔线
- 红色 vertical 标题钉（3-6px × 1em，主标题左侧）
- 顶部色条（navy 或 red）
- 单一几何色块（直角，不允许圆角 > 4px）

## 共通硬规则

无论哪个尺寸，封面必须遵守：

1. **一张封面 = 一个结论**——主标题必须是结论句（"动作可释放 18-24% EBITDA"），不是描述句（"EBITDA 分析"）
2. **主标题字数控制**：21:9 / 16:9 ≤ 14 字 · 1:1 ≤ 10 字 · 3:4 / 9:16 ≤ 8 字
3. **字号阶梯只有 3 档**——主标题 / 副标题（可选） / 元数据（日期 / 编号 / 出处），不允许 4 档以上
4. **避开平台 UI 元素**——参考"安全区位置"列，在那个区域留出 ≥ 15% 的空白
5. **禁用装饰**——不允许渐变 / 阴影 / 圆角 > 4px / WebGL / 点阵 / 3D / 霓虹 / 玻璃拟态
6. **禁用 stock photo**——商务握手 / 团队会议 / 城市俯瞰云端 / AI 机器人 一律禁止
7. **文字必须真实可读**——禁止 lorem ipsum、乱码、扭曲变形字、伪汉字
8. **不要画水印 / 边框 / 二维码 / 平台 logo / 笔电 mockup**——这些交给发图工具自动加
9. **红色 accent 全封面只用 1 处**——通常是标题钉、关键数字、或单一色块
10. **不要画"麦肯锡"字样或任何品牌名**——纯粹是视觉风格，不是 logo

## Prompt 模板（直接喂 GPT-Image-2 / Nano Banana / Imagen）

### 通用模板

```text
生成一张 [21:9 / 1:1 / 3:4 / 16:9 / 9:16] 比例的专业咨询风封面，主题：[主题]。

视觉参考 McKinsey / BCG / Bain 客户汇报材料的封面：
- 背景：纯白 #ffffff（或深色变体 #0a0e13，由内容性质选）
- 主标题：超大无衬线中文字（Inter / Helvetica / 苹方气质），
  字重 700，颜色 #17212B（亮底）或 #ffffff（暗底），
  占画面 [适合比例] 高度，左对齐
- 主标题左侧加一根 [3-6px × 主标题字高] 的红色 vertical rule（#C62828），
  作为唯一红色 accent
- 副标题（可选）：无衬线 + 字重 400 + 颜色 #5C6773，
  字号是主标题的 1/3，与主标题左对齐，行距宽松
- 元数据：等宽小字（IBM Plex Mono / JetBrains Mono），uppercase letter-spacing，
  颜色 #5C6773，位于 [左下 / 右上]，包含 [日期 / 项目编号 / 章节]
- 装饰：1px 极细 hairline 横线（#D9DEE5）做主标题与元数据的分隔
- 网格：12 列 grid，所有元素左对齐，大幅留白做非对称美学
- 安全区：[按尺寸映射]

主标题文字内容：[实际标题，不超过 X 个中文字]
副标题文字内容：[可选，不超过 Y 个中文字]
元数据文字内容：[如 "2026-05-25 · CONFIDENTIAL"]

禁止：渐变、阴影、圆角 > 4px、3D、霓虹、玻璃拟态、卡通、SaaS 模板感、
WebGL 流体、点阵、装饰边框、水印、二维码、笔电 mockup、商务握手照片、
AI 机器人、城市俯瞰云端、stock photo、假 logo、麦肯锡字样、品牌名。
```

### 安全区映射

| 比例 | 主标题位置 | 安全区留白 |
|---|---|---|
| 21:9 公众号头图 | 主标题靠左占 60-70%，红色钉在最左 | 右下 15% 留白避水印 |
| 1:1 公众号分享卡 | 主标题占中央 60%，红色钉在标题左 | 四周各留 10% |
| 3:4 小红书封面 | 主标题靠上占 60-70%，红色钉在标题左 | 底部 25% 留白避用户名 + 标签 |
| 16:9 视频号 / B 站 / YouTube | 主标题居左占 60%，红色钉在最左 | 右下 15% 留白避时长水印 |
| 9:16 抖音 / 短视频竖封 | 主标题在上 1/3，红色钉在最左 | 底部 1/4 留白避进度条 + 操作 UI |

### 深色变体（用于强调 / 收尾 / 章节）

把模板里"亮底"全部反向：
- 背景 `#0a0e13`
- 主标题 `#ffffff`
- 副标题 `rgba(255,255,255,0.65)`
- 元数据 `rgba(255,255,255,0.5)`
- 红色 accent / hairline 不变

深色变体适合 manifesto / 强烈观点 / 大结论；亮底适合常规分享 / 数据汇报。一份多平台封面**全部用同一种底色**（不要一张亮一张暗，视觉不统一）。

## 工作流（给 Agent 的步骤）

1. **问清楚**：用户要哪个平台？要不要一起做多平台？（一次最多做 4 个尺寸）
2. **抽标题**：从文章 / deck 抽 1 句**结论性**主标题，按尺寸控制字数
3. **选底色**：亮底（默认）vs 深底（强烈观点 / 收尾时用）
4. **选 accent 落点**：红色钉用在标题左？还是关键数字？还是单一色块？整张封面只用 1 处
5. **生成**：按 Prompt 模板调用 GPT-Image-2 / Nano Banana / Imagen，**一个尺寸一次调用**
6. **保存**：`cover-{平台}-{尺寸}.jpg`，例如 `cover-gzh-21x9.jpg` / `cover-xhs-3x4.jpg`
7. **自检**：对照下方 checklist

## 自检 checklist

- [ ] 比例匹配平台规格表，**没有自定义比例**
- [ ] 主标题字数符合限制（21:9/16:9 ≤ 14 · 1:1 ≤ 10 · 3:4/9:16 ≤ 8）
- [ ] 主标题是**结论句**，不是"XX 分析"这种描述
- [ ] 字号阶梯只有 3 档（主标题 / 副标题 / 元数据）
- [ ] **全程无衬线**——任何衬线字体出现都是错的
- [ ] **红色只用 1 处**——标题钉 / 关键数字 / 单一色块，三选一
- [ ] **直角到底**——没有圆角 > 4px、没有渐变、没有阴影、没有 WebGL
- [ ] 安全区留白足够，不会被平台 UI 遮挡
- [ ] **没有** 水印 / 二维码 / 平台 logo / 边框 / mockup / 商务握手
- [ ] **没有** 乱码 / 假英文 / 扭曲字 / AI 机器人 / 霓虹
- [ ] 多平台同主题：主标题 + 主题色 + 视觉锚点保持一致
- [ ] 输出像素 ≥ 平台建议值
- [ ] 没有写入 "alanppt"、"麦肯锡"、"BCG" 等任何品牌名

## 路径协同

如果用户**同时需要 deck + 封面**：
- 先做 deck（HTML 或 PPTX）确定主标题和主题色
- 再做封面，主标题**直接复用 deck 第 1 页的结论标题**（裁短到字数限制内）
- 这样 deck 和封面是一体的，发出去有"系列感"

如果用户只要封面，没有 deck：
- 按文章核心结论抽主标题，不需要先做 deck

封面之间也讲究系列感：
- 同一主题做 4 个平台尺寸 → 4 张都用同一主标题（按字数限制裁短）+ 同一底色 + 同一红色 accent 落点
- 不同主题的封面**不要追求"系列感"**，每个主题独立构图即可
