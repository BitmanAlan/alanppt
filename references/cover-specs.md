# 多平台封面规格 · Cover Specs

用于基于一份文章 / PPT 核心观点生成**独立单张封面**(不是 deck 里的一页 slide)。常见落位:公众号头图、公众号分享卡、小红书封面、视频号横版封面。

封面和 PPT 共用同一套视觉系统(Style A 电子杂志风 / Style B 瑞士国际主义),但**绝不要直接截 deck 的某一页当封面**——deck 的横向版式信息密度低,缩到 1:1 或 3:4 后会留白过多、字号失衡。封面必须**重新构图**。

## 何时走这条路径

- 用户说"做一张封面 / 头图 / 分享卡 / 小红书图 / 视频号封面"
- 用户已经有一份 deck 或一篇文章,需要一张配套发图
- 用户要在同一主题下生成多个尺寸的封面(发多个平台)

**不适用**:封面里要嵌很多文字段落(那是海报,不是封面);需要可编辑分层 PSD;需要动图。

## 平台规格表

| 平台 | 比例 | 像素建议 | 安全区位置 | 典型用途 |
|------|------|---------|-----------|---------|
| 公众号头图 | **21:9** | 2400 × 1029 | 顶部 / 左侧,**避开右下角公众号水印** | 顶部 banner |
| 公众号分享卡 | **1:1** | 1080 × 1080 | 中央 60% | 朋友圈 / 群分享缩略图 |
| 小红书封面 | **3:4** | 1080 × 1440 | 上 60%,**避开底部用户名 + 标签条** | 信息流首图 |
| 小红书轮播图 | **3:4** | 1080 × 1440 | 同上,系列内统一字号 | 多图笔记 |
| 视频号横版封面 | **16:9** | 1920 × 1080 | 中央 70%,**避开右下时长水印** | 视频号 / B站 / YouTube 封面 |
| 抖音 / 小红书视频竖封 | **9:16** | 1080 × 1920 | 上 1/3 标题,中部主视觉,**避开底部进度条** | 短视频封面 |

**额外约束**:
- 不允许自定义任意尺寸——只能从上表选
- 同主题多平台时,**视觉主元素 + 主题色保持一致**,只改构图比例和字号
- 像素至少满足建议值,避免上传后压缩糊掉

## 风格继承

| 内容性质 | 走哪套 |
|---------|-------|
| 行业观察 / 人文 / 商业发布 / 演讲海报 | **Style A 电子杂志风**(衬线大标题 + 留白) |
| AI 产品 / 技术发布 / 数据汇报 / 方法论 | **Style B 瑞士国际主义**(无衬线 + 单一 accent) |

主题色从已有的 `themes.md`(A 5 套) / `themes-swiss.md`(B 4 套) 选一套——**不允许临时调色**,和 deck 保持一致最稳。

## 共通硬规则

无论 A 还是 B,封面必须遵守:

1. **一张封面 = 一句话**——主标题不超过 14 个中文字(瑞士风可降到 10 字以内)
2. **字号阶梯只有 3 档**——主标题 / 副标题(可选) / 元数据(日期 / 编号 / 出处),不允许 4 档以上
3. **避开平台 UI 元素**——参考"安全区位置"列,在那个区域留出 ≥ 15% 的空白
4. **不要画水印 / 边框 / 二维码 / 平台 logo**——生成图只负责主视觉,水印交由发图工具自动加
5. **不要堆装饰**——一个主视觉锚点(照片 / 衬线大字 / 数据 / 几何图形)足够,加越多越像营销号配图
6. **文字必须真实可读**——禁止 lorem ipsum、乱码、扭曲变形字、伪汉字
7. **不要带笔记本 / 手机 mockup**——这是 deck 截图风,不是封面风
8. **不要 stock photo 商务摆拍**——握手、白人会议、城市俯瞰云端那套,一秒掉档次

## Style A · 电子杂志风封面

视觉基调:像 *Monocle* / *Kinfolk* / *MIT Tech Review* 的封面那样克制。

### 关键元素

- **大衬线主标题**(Noto Serif SC + Playfair Display),占画面 35-50% 高度
- **元数据等宽字体**(JetBrains Mono / IBM Plex Mono),做编号 / 日期 / 第 N 期
- **纸张质感底**——和 `themes.md` 选定主题的 `--paper` 一致(米白 / 牛皮 / 沙色 / 墨水黑)
- **一张纪实摄影**(可选)——置于安全区外侧,**不能盖标题**
- **细线分割**——1px hairline,不用粗框

### Prompt 模板(可直接喂 GPT-Image 2.0 / GPT-M 2.0)

```text
生成一张 [21:9 / 1:1 / 3:4 / 16:9] 比例的电子杂志风封面,主题:[主题]。
风格参考 Monocle / Kinfolk 杂志封面:大衬线中文主标题居[左上 / 中央],
背景使用 [墨水经典 / 靛蓝瓷 / 森林墨 / 牛皮纸 / 沙丘] 主题的纸张色,
左下或右上放等宽字体的小型元数据(如 "ISSUE 03 · 2026"),
可选一张低饱和纪实摄影作为辅助视觉,放在安全区外侧。
主标题文字内容:[实际标题,不超过 14 个中文字]。
不要画任何边框、水印、二维码、平台 logo、笔电 mockup、商务摆拍照片。
不要 AI 机器人、霓虹、3D、玻璃拟态、SaaS 模板感。
保持极简、克制、印刷品质感。
```

### Style A 安全区映射

- **公众号 21:9**:主标题靠左占 60%,右侧 40% 留给摄影或留白
- **公众号 1:1**:主标题大字占中央 70%,顶部留 15% 放小型元数据
- **小红书 3:4**:主标题靠上,占顶部 60%;底部 25% 留白避开平台 UI
- **视频号 16:9**:主标题居中或左对齐,占左侧 65%,右侧留给摄影或负空间

## Style B · 瑞士国际主义封面

视觉基调:像 Massimo Vignelli 的 Unimark 海报、IKEA 早期 catalog、Acne Studios 季度 lookbook。

### 关键元素

- **超大无衬线主标题**(Inter / Helvetica / Noto Sans SC),**字重 200/300**,占画面 40-60% 高度
- **单一 accent 色块**(IKB 蓝 / 柠檬黄 / 柠檬绿 / 安全橙),做几何图形或文字底
- **网格对齐**——所有元素吸附到 12 列 grid,左对齐 + 大幅留白
- **等宽元数据**——`mono` `uppercase`,如 `2026 · NO.07 · SHANGHAI`
- **直角到底**——任何圆角 / 阴影 / 渐变都是错的(rule 横线除外)

### Prompt 模板

```text
生成一张 [21:9 / 1:1 / 3:4 / 16:9] 比例的 Swiss International Typographic Style 封面,主题:[主题]。
视觉参考 Massimo Vignelli / Josef Müller-Brockmann / Helvetica Forever。
画面使用极简 12 列 grid,左对齐,大幅留白。
主标题用 Helvetica/Inter 气质的无衬线超大中文字(字重 200-300),
颜色为深黑 #0a0a0b 或纯白,占画面 40-60% 高度。
背景纯色:白色 #fafafa 或深黑 #0a0a0b 二选一。
唯一 accent 色为 [IKB 蓝 #002FA7 / 柠檬黄 #FFD500 / 柠檬绿 #C5E803 / 安全橙 #FF6B35],
做一个直角几何色块或文字底,占画面 8-15%。
左下或右上加等宽字体小型元数据(如 "2026 · NO.07")。
主标题文字内容:[实际标题,不超过 10 个中文字]。
禁止:渐变、阴影、圆角、3D、霓虹、玻璃拟态、卡通、SaaS 模板感、假 logo、装饰边框、水印、二维码、笔电 mockup、商务摆拍照片。
```

### Style B 安全区映射

- **公众号 21:9**:超大主标题靠左占 70%,右侧留给 accent 色块 + 元数据
- **公众号 1:1**:主标题居左 + 占中央 60%,右下 accent 小色块做视觉钩子
- **小红书 3:4**:主标题靠上 + 占顶部 70%;底部 25% 留白,可加一行 accent 色短副标
- **视频号 16:9**:主标题大字居左占 50%,右侧 accent 几何 + 等宽元数据

## 工作流(给 Agent 的步骤)

1. **问清楚**:用户要哪个平台?要不要一起做多平台?(一次最多做 4 个尺寸)
2. **风格选**:用户的 deck 是 A 还是 B?没有 deck 就根据内容性质问一次
3. **主题色**:从 `themes.md` / `themes-swiss.md` 挑一套
4. **抽标题**:从文章 / deck 抽 1 句不超过 14 字(A)或 10 字(B)的主标题;副标可选
5. **生成**:按对应 Prompt 模板调用 GPT-Image 2.0 / GPT-M 2.0,**一个尺寸一次调用**
6. **保存**:`cover-{平台}-{尺寸}.jpg`,例如 `cover-gzh-21x9.jpg` / `cover-xhs-3x4.jpg`
7. **自检**:对照下方 checklist

## 自检 checklist

- [ ] 比例匹配平台规格表,**没有自定义比例**
- [ ] 主标题字数控制在 14(A)/ 10(B)个中文字以内
- [ ] 字号阶梯只有 3 档
- [ ] 主题色和 deck 一致(同主题不混色)
- [ ] 安全区留白足够,不会被平台 UI 遮挡
- [ ] **没有**水印 / 二维码 / 平台 logo / 边框 / mockup / 商务摆拍
- [ ] **没有**乱码 / 假英文 / 扭曲字
- [ ] Style B:全程无衬线、单一 accent、直角到底
- [ ] Style A:衬线主标题、纸张质感底、不超过一张辅助摄影
- [ ] 多平台同主题:视觉主元素 + 主题色保持一致
- [ ] 输出像素 ≥ 平台建议值
