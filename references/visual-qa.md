# 视觉 QA · 渲染 + 换眼挑刺回环（交付前必跑）

> 适用于 HTML deck 与 PPTX 两条路径。核心信念：**你的第一版几乎一定有问题。** 你一直盯着自己的代码，只会看到"你以为的样子"，不是"真实的样子"。把 QA 当成 bug 猎杀，不是确认仪式——如果第一遍你一个问题都没找到，是你没认真找。

## 为什么必须换一个子agent

同一个 agent 有"预期盲区"：它知道每个元素**应该**在哪，于是脑补成对的，看不出实际挤了/溢出了。**必须 spawn 一个全新子agent**，给它图、不给它代码，让它用纯视觉视角挑刺。哪怕只有 2-3 页也要换眼。

## Step 1 · 渲染成图

**HTML deck**：
```bash
# 用 headless Chrome 逐页截图（或 playwright）。每页 1600×900 截图到 /tmp/qa/
# 例：node 脚本里对每个 .slide.active 截图，或用浏览器打印 PDF 再 pdftoppm
```
若无脚本，至少在浏览器逐页打开、用 ⌘⇧4 截全屏存图。

**PPTX**：
```bash
python "<SKILL_ROOT>/assets/mckinsey-pptx/scripts/office/soffice.py" --headless --convert-to pdf deck.pptx 2>/dev/null \
  || soffice --headless --convert-to pdf deck.pptx
pdftoppm -jpeg -r 150 deck.pdf /tmp/qa/slide
```

## Step 2 · 派子agent挑刺（用这段 prompt）

```
视觉检查这些幻灯片。假设里面有问题——把它们找出来。逐页排查：

【呼吸 / 留白】（alanppt 最易翻车，重点查）
- 有没有"喘不过气"的区域：某处空旷、某处拥挤，疏密不均？
- 区块之间贴得太近（间距 < 画布 3%）？卡片/元素几乎挨到一起？
- 内容顶到边缘（距版边 < 画布 6%）、压住页脚 hairline 或页码？
- 整页负空间是否 < 30%（显得满、闷）？是否该删一点 / 拆一页 / 加留白？
- 留白是否"分布"而非"塌底"：内容是否堆在上半部、下半页大片空白显得没做完？→ 给正文区加 `.center-y` 纵向居中，或补一行论据让上下平衡
- 文本块是否太宽顶到边（中文 > 34 字一行）、或太窄导致频繁换行？

【硬伤】
- 元素重叠（文字穿过色块、线压字、堆叠）
- 文字溢出或被边界截断
- 标题钉/色条是按单行标题设计的，但标题换行成两行后错位
- 列与列、卡与卡没有对齐一致
- 低对比文字（浅灰字压在浅底上）、深底上的深色图标无对比圈

逐页列出所有问题（含轻微的）。每页给：Expected（一句预期）+ 实际发现的问题。
图片：
1. /tmp/qa/slide-01.jpg（Expected: 封面，结论标题）
2. ...
```

## Step 3 · 修复 → 重验回环

1. 列出全部问题（一个都没找到 → 用更挑剔的眼光再看一遍）
2. 修
3. **重新渲染受影响的页**——一处修复常引出另一处问题
4. 重复，直到完整一轮零新问题
5. **没跑完至少一轮"修复+重验"，不准宣布完成**

## 与既有 checklist 的关系

`checklist-mckinsey.md`（pptx）/ checklist（html）是**静态自查**（颜色、字号、可编辑等），本回环是**动态视觉验证**。两者都要过：先自查，再换眼渲染 QA。
