# Rebuild Prompt · 把 PNG 素材重建成可编辑 pptx

> 用法：本 Prompt 是给**多模态对话模型**（如 ChatGPT / Claude）的兜底方案——
> 当你不想或不能在本地跑 Node 脚本时，可以把 PNG 素材打包发给模型，让它生成 pptx。
> **推荐做法仍然是用本地脚本**：见 `assets/mckinsey-pptx/make-deck.js`，质量稳定可控、可重复。
> 详细工作流见 `references/mckinsey-pptx.md`。

---

## Prompt（中文版 · 直接复制发给模型）

```text
请使用刚才拆分得到的所有 PNG 素材，重新还原一套完整的可编辑 PPT 文件。

要求：
1. 按照原始页面布局，将所有 PNG 素材放回对应位置，还原为完整页面。
2. 所有图片保持原比例和位置关系。
3. 不要把文字做成图片。
4. 在图片基础上加入可编辑文本框，文字为独立可编辑内容。
5. 整体输出为一个可编辑的 .pptx 文件，16:9 比例，共 N 页。
6. 页面应保留麦肯锡/咨询汇报风格：结论先行、网格清晰、信息密度高、颜色克制。

目标：
最终得到一个可以直接修改文字、调整布局、替换图表元素的 PPT 文件。
```

## 为什么推荐本地脚本而不是模型生成

| 维度 | 模型直接生成 pptx | 本地 `make-deck.js` |
|---|---|---|
| 文字精度 | 中文长内容可能漏字 | 100% 精确（来自 `addText` 字符串） |
| 图层位置 | 像素级偏差 5-15px | 像素级精确（脚本写死坐标） |
| 中文字体 | 可能 fallback 到错误字体 | 显式 `fontFace: "Arial"`，PowerPoint 自动 fallback |
| 红色/绿色等 accent | 模型可能配错 | 写死 `C.red / C.green` 常量 |
| 可重复 | 每次结果不一样 | 同样输入 → 同样输出 |
| 修改成本 | 微调要重发 Prompt | 改脚本 +重跑（几秒） |
| 适合场景 | 一次性 demo / 没本地环境 | 客户交付 / 多轮迭代 |

**结论**：客户项目走 `make-deck.js`。模型直出 pptx 只在以下场景考虑：
- 你在用 web 版 ChatGPT/Claude，没装 Node
- 一次性 demo，不需要二次修改
- 5 页以内的简单 deck
